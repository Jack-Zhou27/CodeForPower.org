LESSON 1: PRIMITIVE DATA TYPES:
What is data?


Data is information, such as numbers,
conditional statements (yes, no),
and more. For computers to process
such information, data types and data
structures exist to store them. (
NOTE* computers don’t understand data.
They just have access to the ones you store.)


Primitive data types:
There are 8 primitive data types that
are predetermined and built into Java.
All primitive data types have a
predetermined memory size allocated
for their storage.
The Primitive Data Types (8)
Listed below are the 8 primitive data types in Java.
NOTE: floating-point types should never be used
to store exact values due to accuracy errors.

int (Integral primitive type)
Ints will be the most frequently used
primitive data type that one will use
to store an integer (such as 2,100,467).
Ints are among the most frequently used
primitive types, and therefore, should
be one that is thoroughly understood in
its applications. You cannot store ints
over their upper and lower boundaries. 
NOTE: the int can overflow should 
you perform arithmetic with integers 
that store the largest or smallest 
numbers that it can store. For example, 
if you print “value”, an int that stores 
2,147,483,647, and add one to it,
it will overflow and print the smallest
possible integer that the int can store 
(will print -2,147,483,648 instead).
The following formulas can be used
to see what is going to be the output
should you overflow a int intentionally 
or purposefully. Integer values, as of
Java SE 7, can have underscores 
between its digits for readability.
(example: 99_012)

(int value = 2,147,483,647): System.out.print(value + n) →  -2,147,483,648 + n - 1
(int value = -2,147,483,648): System.out.print(value - n) →  2,147,483,648 - n + 1



double (floating-point type)
Doubles are another frequent data 
type that Java programmers use to 
store decimals (3.1415926). Doubles 
are also another frequently used
primitive data type, and therefore,
should be understood in its 
applications as well. Typically, 
any decimal will be expressed as 
a double instead of a float as memory
size isn’t a problem that concerns 
modern day computers. Doubles are
also more powerful than ints as they
can store ints. In doing so, the integer 
stored will be promoted into a double
through the appending of zeros after
the units digit. (32.000) This can, 
however, result in accuracy errors 
(32.001) and should be used cautiously. 
Doubles, therefore, should never be 
used to store precise values. Values 
exceeding its range cannot be stored 
as a double.
Syntax for primitive data declarations (primitive type) dataName = (data that the type can store); NOTE: (primitive type = int, double, boolean, byte, short, long, float) char dataName = ‘data that char can store’;
